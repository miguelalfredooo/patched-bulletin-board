## Issue & Fix: Multi-Agent Migration
**Problem:** After setting up multi-agent system, got error: "Agent 'main' no longer exists in configuration"

**Root Cause:** Migrating from single-agent (`main`) to multi-agent (`coda`, `maeve`, `victor`) leaves old session state in `~/.openclaw/agents/main/` that the gateway tries to load

**Fix Applied:**
```bash
rm -rf ~/.openclaw/agents/main
openclaw gateway restart
```

**Result:** Gateway now correctly routes to default agent (Coda) without session conflicts

**Documentation:** Added to CLAUDE.md § Gotchas & Troubleshooting
