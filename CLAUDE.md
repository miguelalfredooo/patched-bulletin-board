## Full Setup (The Hard Way — So You Don't Have To)

### Step 1: Anthropic API Key (Critical)

Get your API key from https://console.anthropic.com/

Add to `~/.zshrc`:

```bash
export ANTHROPIC_API_KEY="your-actual-key-here"
```

**Why this matters:** Every time you start a new shell, the API key must be available in the environment. OpenClaw's auth system reads it from `$ANTHROPIC_API_KEY`.

### Step 2: Gateway Configuration — `~/.openclaw/openclaw.json`

This file is **the root of all OpenClaw configuration**. It must be perfectly formed:

```json
{
  "gateway": {
    "mode": "local",
    "auth": {
      "mode": "token",
      "token": "your-gateway-token-here"
    }
  },
  "agents": {
    "defaults": {
      "workspace": "/Users/blackmachete/openclaw-artifacts",
      "model": {
        "primary": "anthropic/claude-sonnet-4-6"
      }
    }
  },
  "auth": {
    "profiles": {
      "anthropic:default": {
        "provider": "anthropic",
        "mode": "api_key"
      }
    }
  },
  "plugins": {
    "entries": {
      "anthropic": {
        "enabled": true
      },
      "discord": {
        "enabled": true
      },
      "bonjour": {
        "enabled": false
      }
    }
  },
  "channels": {
    "discord": {
      "enabled": true,
      "token": "your-discord-bot-token-here"
    }
  }
}
```

**Critical fields:**
- `gateway.mode`: MUST be `"local"` (not a string like "local-mode", just `"local"`)
- `gateway.auth.token`: Generated by OpenClaw or manually created. Get it from `openclaw config get gateway.auth.token`
- `agents.defaults.workspace`: MUST point to your agent workspace (`/Users/blackmachete/openclaw-artifacts`)
- `agents.defaults.model.primary`: MUST be `"anthropic/claude-sonnet-4-6"` (the Anthropic Sonnet model)
- `auth.profiles.anthropic:default.mode`: MUST be `"api_key"` (reads from env var)
- `plugins.entries.bonjour.enabled`: MUST be `false` (bonjour causes unhandled promise rejections that crash WebSocket connections — this was the final blocker)

### Step 4: Agent Auth Store

OpenClaw maintains isolated auth stores for each agent. The agent needs access to your Anthropic credentials.

Create/update `~/.openclaw/agents/main/agent/auth-profiles.json`:

```json
{
  "anthropic:default": {
    "provider": "anthropic",
    "mode": "api_key",
    "apiKey": "your-actual-api-key-here"
  }
}
```

**Why:** Isolated agents can't read the global gateway auth by default. They need their own credential store.

### Step 5: Start the Gateway

```bash
# Ensure env var is set
source ~/.zshrc

# Kill any stray processes (critical if you've restarted)
pkill -f "openclaw-gateway"
sleep 2

# Start the gateway
export ANTHROPIC_API_KEY="your-key-here"
openclaw gateway &
```

Wait 4–5 seconds. Check:

```bash
ps aux | grep openclaw | grep -v grep
```

Should see `openclaw-gateway` running.

## Gotchas & Troubleshooting

### "All models failed" / "No API key found"

**Root cause:** Agent's isolated auth store is empty or malformed.

**Fix:**
1. Verify `~/.openclaw/agents/main/agent/auth-profiles.json` exists
2. Check it has correct format (provider + mode + apiKey)
3. Restart gateway: `pkill -f openclaw-gateway && sleep 2 && openclaw gateway &`

### "Gateway closed (1006)" / WebSocket disconnections

**Root cause:** Bonjour plugin unhandled promise rejection. It tries to advertise the gateway via mDNS, fails silently, and crashes the WebSocket.

**Fix:** Disable bonjour in `openclaw.json`:
```json
"bonjour": {
  "enabled": false
}
```

Restart gateway.

### Port 18789 already in use

**Root cause:** Previous gateway process still running (often as LaunchAgent background service on macOS).

**Fix:**
```bash
lsof -i :18789  # See what's using it
pkill -f openclaw-gateway
sleep 2
openclaw gateway &
```

Or stop the LaunchAgent:
```bash
openclaw gateway stop
```

### "Gateway not reachable" after startup

**Root cause:** Gateway crashed during initialization. Usually auth or plugin errors.

**Fix:**
```bash
# Check logs
openclaw logs --follow

# Or run diagnostics
openclaw doctor

# Restart with verbose output to see what's failing
export ANTHROPIC_API_KEY="your-key-here"
openclaw gateway --verbose 2>&1 | head -100
```

### Workspace not loading (bootstrap files not injected)

**Root cause:** `agents.defaults.workspace` is wrong or bootstrap files don't exist.

**Fix:**
1. Verify `openclaw.json` has: `"workspace": "/Users/blackmachete/openclaw-artifacts"`
2. Verify all 4 bootstrap files exist in repo root:
   - IDENTITY.md
   - SOUL.md
   - AGENTS.md
   - USER.md
3. Restart gateway

### Discord token not working

**Root cause:** Token is stale or malformed, or the bot doesn't have permissions in the target channel.

**Fix:**
1. Generate a fresh Discord bot token
2. Update `openclaw.json` channels.discord.token
3. Ensure bot has "Send Messages" permission in target channel
4. Restart gateway

## Architecture

```
~/.zshrc (env var)
    ↓
$ANTHROPIC_API_KEY (in memory)
    ↓
~/.openclaw/openclaw.json (global config)
    ↓
    ├→ gateway.auth.token (gateway security)
    ├→ agents.defaults.workspace (where Max lives)
    ├→ agents.defaults.model (which LLM to use)
    ├→ auth.profiles.anthropic:default (auth method: api_key)
    ├→ plugins (which subsystems to load)
    └→ channels.discord (bot integration)
    ↓
openclaw gateway (HTTP + WebSocket server on 18789)
    ↓
Max Arias agent (in workspace)
    ↓
    ├→ IDENTITY.md (name, role, emoji)
    ├→ SOUL.md (voice + stance)
    ├→ AGENTS.md (operating rules)
    └→ USER.md (Alfred profile)
    ↓
WebChat UI (http://127.0.0.1:18789/)
```

## Never Commit

- `~/.openclaw/openclaw.json` — contains gateway tokens and API keys
- Session transcripts (in `.openclaw/agents/main/sessions/`)
- Channel state files
- `auth-profiles.json`
- `.env` files with actual keys

The `.gitignore` in openclaw-artifacts handles project files. Never commit the global `~/.openclaw/openclaw.json`.
