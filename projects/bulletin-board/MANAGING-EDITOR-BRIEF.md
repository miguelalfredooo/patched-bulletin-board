## File Format

**Example:**

```markdown
**Art — Specificity as Clarity**

When artists compress complexity into constraint, something clarifies. Not simplification—compression. The difference: simplification loses information; compression holds it all, just tighter. This week's art explores the compression of ideas into form. What happens when you force an idea through an impossibly small opening? It comes out sharp.

https://example.com/source1
https://example.com/source2
```

**Format rules:**
- Title line: `**Section Name — Subtitle**`
- Raw markdown (no backticks, no escaping)
- 60-120 words
- 1-2 URLs
- No placeholder text

## Apartamento Register

The voice should be:
- **Intimate** (like talking to a friend, not lecturing)
- **Specific** (concrete examples, not abstractions)
- **Unhurried** (take time to explain, don't rush)
- **No corporate language** (no "innovative," "cutting-edge," "synergy")

**Example voice:**

❌ **Wrong (corporate):** "This innovative approach leverages cutting-edge methodologies to enhance visual communication across multiple platforms."

✅ **Right (Apartamento):** "The designer took a simple idea—that constraints make things clearer—and pushed it until clarity broke. What emerged was messy. Specifically interesting."

**Section copy (isolated code blocks):**
For each of 11 sections, send:
- Section header (e.g., "Section 1: Art")

- Brief note about the section

**Closing observations (required):**
```
Observations (required):
- Theme clarity: [Was the Editorial Brief clear? Any ambiguities?]
- Voice challenge: [What was hardest about maintaining consistent voice?]

- Art direction: [What visual approach would best complement these sections?]

```

### Code Block Rules

- **Each section:** Gets its own code block (isolate from other sections)
- **No combining:** Do not merge multiple sections into one block
- **No splitting:** Do not break a section across multiple blocks
