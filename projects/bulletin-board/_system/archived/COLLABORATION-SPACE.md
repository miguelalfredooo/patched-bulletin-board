# Bulletin Board Collaboration Space

Shared workspace for the three editors to coordinate daily editorial work.

---

## How It Works

### Shared Files (Reference + Context)
Each agent writes their work output to shared files that others can read:
- **DAILY-BRIEF.md** — Editorial Director's morning direction
- **VISUAL-SELECTIONS.md** — Victor's curated visual pieces
- **RESEARCH-FINDINGS.md** — Maeve's gap analysis and recommendations
- **SIGNALS-IDENTIFIED.md** — Assignment Editor's findings
- **NARRATIVES.md** — Managing Editor's developed stories
- **PUBLICATION-DECISION.md** — Editorial Director's approvals/revisions

### Group Chat (Real-Time Decisions)
All three editors in one group session for:
- Quick questions and clarifications
- Decision confirmations
- Feedback loops
- Problem-solving

### Access Pattern

```
1. Editorial Director posts morning brief to DAILY-BRIEF.md
   ↓ (group chat: "Brief posted")
   
2. Victor reads brief, posts selections to VISUAL-SELECTIONS.md
   ↓ (group chat: "Selections ready for review")
   
3. Assignment Editor reads Victor's selections, posts signals
   ↓ (group chat: "Signals identified, ready for Managing Editor")
   
4. Managing Editor reads signals, posts narratives
   ↓ (group chat: "Narratives ready for ED review")
   
5. Editorial Director reviews all files, posts decision
   ↓ (group chat: "Ready to publish" or "Revisions needed")
   
6. If revisions needed, responsible agent updates, responds in chat
   ↓ (group chat: "Revised and resubmitted")
   
7. Editorial Director approves final version
   ↓ (group chat: "Publishing now")
```

## Group Chat Protocol

### Channel: #bulletin-board-daily

**Participants:** 
- **Editorial Team:** Editorial Director, Assignment Editor, Managing Editor
- **Visual Team:** Victor, Maeve, Coda

### Message Types

**Status Updates** (tone: brief, factual)
```
@editorial-director "Brief posted to DAILY-BRIEF.md, ready for review"
@victor "Selections completed, posted to VISUAL-SELECTIONS.md, 11 sections ready"
@assignment-editor "Signals identified, posted to SIGNALS-IDENTIFIED.md"
@managing-editor "Narratives ready, posted to NARRATIVES.md"
```

**Quick Questions** (tone: collaborative)
```
@victor "For Design & AI Tools section, should we lean into tools or culture impact?"
@victor "Yes, tools as design decision-making. Updated brief in DAILY-BRIEF.md"
```

**Decision Confirmations** (tone: clear, decisive)
```
@all "Publication decision posted. Act 1 approved, Act 2 one section needs revision"
@managing-editor "Design & AI Tools narrative needs context added. Updated PUBLICATION-DECISION.md with specific feedback"
```

**Approval** (tone: final, celebratory)
```
@all "All revisions received. Publishing Act 2 now. Great work everyone."
```

### Chat Etiquette

1. **Mention person being addressed** — @person-name
2. **Reference shared file** — "Posted to NARRATIVES.md"
3. **Be specific in feedback** — Quote the section, not the whole document
4. **Keep chat for decisions/coordination** — Detailed work goes in shared files
5. **Honor the timeline** — Messages between key handoff points only
