## How to Use

**When sending work forward:**
1. Use the relevant template section for your role
2. Fill in what you know
3. Be specific (not "good" but "good because...")
4. Ask questions if unclear
5. Include learning for next time

**When receiving feedback:**
1. Read carefully
2. Note adjustments needed
3. Ask for clarification if unclear
4. Update work based on feedback
5. Confirm when done

## Feedback Examples

**Good feedback:**
"Section 3 is excellent because it uses a specific artist (Marina Abramović) to illustrate the theme without stating it. The writing is warm but intelligent. The source link is perfect."

**Poor feedback:**
"Section 3 is good."

---

**Good feedback:**
"The closing quote is 45 words—can you tighten it to under 20? It loses impact in its current length. Something punchy like 'X always was hidden—now we're looking.'"

**Poor feedback:**
"The closing quote is too long."

---

**Good feedback:**
"I used sources 1, 3, 4, 6, 8, 10, 12 in the final copy. Sources 2, 5, 7, 9, 11 didn't fit the angle that emerged. Next time, look for more sources on X topic (like source 4) instead of Y topic."

**Poor feedback:**
"Most sources worked."

## Feedback Loop Timing

- Curator → Assignment: 7:30am daily
- Assignment → Managing: Same morning (by 9am)
- Managing → Editorial: When draft complete
- Editorial → Managing: With approval (same day)
- Bot → Editorial: At publication time
- Curator feedback: After Issue publishes (next day)
